# HM Goal & Metrics

Назначение
- Стандартизированная постановка цели и метрик, включая балансирующие, с проверками на типовые ошибки.

Когда применять
- В начале стратегической работы или при ревизии существующих целей/метрик.

Что уточнить у пользователя
- Краткое описание цели (без чисел внутри текста цели).
- Метрики: список, текущее значение, целевое значение, тип (опережающая/запаздывающая).
- Балансирующие метрики: что нельзя ухудшить, целевые пороги.

Выходной формат
```
Цель: <описание>
Метрики:
- <metric_name>: current=<...>, target=<...>, type=leading|lagging
Балансирующие метрики:
- <metric_name>: hold >=|<= <...>
```

JSON
```
{
  "goal": {"name": "...", "description": "..."},
  "metrics": [
    {"name": "...", "current": "...", "target": "...", "type": "leading|lagging"}
  ],
  "balancing_metrics": [
    {"name": "...", "constraint": ">=|<=", "value": "..."}
  ]
}
```

Правила валидации
- Цель — не список задач и не бинарные артефакты («создать дашборд»). Числа — в метрики.
- Метрики измеримы и позволяют однозначно судить о достижении цели.
- Балансирующие метрики фиксируют «не ухудшить» и имеют понятные пороги.

Типовые ошибки и авто‑подсказки
- Задача вместо цели: переписать цель как желаемый результат/состояние.
- Бинарные метрики: заменить на количественные показатели влияния.
- Недостаток метрик: предложить 2–3 релевантных, разделив на leading/lagging.

Связанные топологии
- Метрики цели на отдельном уровне; иерархия целей; многоцелевая карта.
