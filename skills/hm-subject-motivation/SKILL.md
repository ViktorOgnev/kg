# HM Subject & Motivation

Назначение
- Описание субъекта(ов), их болей/желаний и ключевых мотиваций для увязки гипотез.

Когда применять
- Перед формулировкой via_subject гипотез или при проверке «потому что» в существующих гипотезах.

Что уточнить у пользователя
- Список субъектов (роль/группа), краткое описание.
- Боли/желания по каждому субъекту, приоритеты.
- Есть ли негативные субъекты (и их намерения/влияния).

Выходной формат
```
Субъект: <name> — <описание>
Боли/желания: ["...", "..."]
Негативный субъект?: yes|no (описание влияния)
```

JSON
```
{
  "subjects": [
    {"name": "...", "about": "...", "pains_desires": ["..."], "negative": false}
  ]
}
```

Правила валидации
- Субъект обладает самодвижением (не процесс/не абстрактный объект, не «исполнитель вместо субъекта»).
- Мотивации релевантны и соотносятся с «потому что» в гипотезах.

Типовые ошибки и авто‑подсказки
- Исполнители вместо субъектов: отделить роли исполнителей от субъектов влияния.
- «Потому что» не о субъекте: скорректировать, опираясь на боли/желания.
- Не ключевая мотивация: выделить корневые мотивы, остальные в заметки.

Топологии
- Мотивация субъекта на отдельном уровне.
