# HM Orchestrator

Назначение
- Провести пользователя сквозным сценарием создания карты гипотез, не перегружая вопросами. Каждый шаг задаёт один ключевой вопрос, затем короткий чек и переход к следующему шагу. Итог: структурированные данные карты + визуализация.

Принципы
- Один экран — один шаг: максимум 1 главный вопрос + до 3 уточнений.
- Мини‑результат на шаге: краткий вывод ≤10 строк и «Что дальше?». 
- Автодозапрос недостающих данных. Разумные дефолты — явно помечать как [предложение].
- Состояние процесса хранить на диске, чтобы можно было прервать/возобновить.

Хранилище состояния
- `.hm/session.json` — прогресс мастера (текущий шаг, чек‑лист и т.п.).
- `.hm/map.json` — канонические данные карты (цель/метрики/субъекты/гипотезы/задачи/приоритеты/связи/заметки/блокеры). См. схему в `hm-json-export`.
- При каждом шаге: читать обе структуры; после шага — сохранять обновлённые.

Шаги (сквозной сценарий)
0) Инициализация
   - Если `.hm/map.json` отсутствует — создать каркас с пустыми секциями.
1) Цель и метрики (use_skill: hm-goal-metrics)
   - Спросить только то, чего нет. На выходе: цель, 1–2 метрики, 1–2 балансирующие.
2) Субъекты и мотивации (use_skill: hm-subject-motivation)
   - Предложить 3–5 субъектов по контексту; спросить «оставить/исключить/добавить». 
3) Гипотезы (use_skill: hm-hypothesis-builder)
   - На каждого субъекта: 1–2 via_subject гипотезы. По желанию — 1 direct на цель. 
   - Прогнать проверки типовых ошибок; короткий чек и точечные правки.
4) Задачи (use_skill: hm-task-linker)
   - Превратить гипотезы в задачи с жёсткой привязкой; «нет привязки — нет задачи».
5) Приоритизация (use_skill: hm-prioritization)
   - Назначить high/mid/low и выделить приоритетный путь (пояснить «почему»).
6) Экспресс‑ревью (use_skill: hm-review-troubleshooter)
   - Выдать 3–7 правок; спросить, что применить автоматически.
7) Экспорт (use_skill: hm-json-export)
   - Обновить `.hm/map.json` (и YAML, если попросили). 
8) Визуализация
   - Fallback: сгенерировать `hm/map.mmd` (Mermaid flowchart) из `.hm/map.json`.
   - Интеграции (опционально, поздние стадии): MCP для Miro/Holst — разместить элементы с цветами стандарта.

Форматы вывода
- Мини‑результат шага: краткий текст + bullets «Проверить/исправить/далее».
- Итог: `.hm/map.json` + `hm/map.mmd` (если пользователь запросил визуализацию).

Mermaid визуализация (fallback)
- Использовать flowchart TB. Определить classDef под стандартные цвета «Карты гипотез»:
  - goal/metrics: `#cadf58`, subject: `#ffc831`, neg.subject: `#ffb373`, hypothesis: `#ffef73`, task: `#a6cdff`, note: `#f1f1f1`, blocker: `#FFA391`. Связи серые `#70736D`. Приоритет окрашивать цветом стрелки (минимум: добавить классы links).
- Скрипт-конвертер: `scripts/hm_map_to_mermaid.py` — читает `.hm/map.json`, пишет `hm/map.mmd`. 
- Рекомендации по визуальному стилю см. `skills/shared/references/mermaid-style.md` (вдохновлено pretty‑mermaid‑skills).

Интеграция с узкими скиллами
- В каждом шаге при необходимости вызывать: `use_skill("hm-…")`.
- Перед вызовом — подготавливать минимальные входы; после — вносить правки/сведение в `.hm/map.json`.

Управление потоком диалога
- На каждом шаге предлагать выбор: [Далее] [Уточнить данные] [Исправить пункт N] [Завершить сейчас].
- Любая новая вводная от пользователя может перепрыгнуть к соответствующему шагу (напр., «добавь субъекта HR»).

Завершение
- Напомнить «Главное правило»: ресурсы выделяются только на задачи из Карты гипотез.
- Предложить сохранить снапшот (копию) `.hm/` и `hm/map.mmd`.
