# Набор скиллов для «Карты гипотез»

Этот каталог содержит модульные скиллы для составления и сопровождения сущностей «Карты гипотез». Язык: русский. Формат совместим с подходом skill-creator (Anthropic).

Состав
- hm-hypothesis-builder — оформление и проверка гипотез (ядро)
- hm-goal-metrics — формулировка цели, метрик и балансирующих метрик
- hm-subject-motivation — субъект, мотивации, негативные субъекты
- hm-task-linker — генерация и привязка задач к веткам
- hm-review-troubleshooter — экспресс‑аудит карты и план фикса
- hm-topology-advisor — рекомендации по топологиям представления (доп.)
- hm-prioritization — приоритизация и приоритетный путь (доп.)
- hm-json-export — экспорт в JSON/YAML (Strategy as Code) (доп.)
 - hm-orchestrator — пошаговый фасилитатор: ведёт от цели до визуализации, хранит состояние, генерирует Mermaid

Общие материалы
- shared/references/hypothesismapping-terms.md — краткие термины/напоминания
- shared/references/common-checklists.md — мини‑чеклисты
 - shared/references/mermaid-style.md — принципы оформления Mermaid для HM

Как использовать локально
1) Положите каталог `skills/` в ваш репозиторий (уже сделано).
2) Если у вашей среды есть механизм загрузки скиллов — укажите путь к нужному SKILL.md. Иначе используйте их как подробные инструкции/шаблоны.

Публикация через git
- Репозиторий уже redistributable: каждый скилл — самостоятельная папка с `SKILL.md`.
- Рекомендуется версионировать изменения и сопровождать их примерами.

Упаковка (опционально, если используется skill-creator)
- Для генерации `.skill` пакета воспользуйтесь скриптами из `anthropics/skills/skill-creator` (см. их README). Общая идея:
  - Подготовьте структуру `<skill-name>/SKILL.md` (+опциональные scripts/references/assets).
  - Упакуйте: `python -m scripts.package_skill skills/<skill-name>`
  - Размещайте получившийся артефакт .skill как релиз.

Замечания
- Тексты скиллов компактны и фокусируются на валидации и связности между элементами карты.
- Экспорт в JSON позволяет использовать «Стратегию как код»: хранить карту в VCS, интегрировать с трекинг‑системами и CI.
