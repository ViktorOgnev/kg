# HM Task Linker

Назначение
- Преобразовать гипотезы в набор проверочных/реализационных задач и жёстко привязать их к веткам карты.

Когда применять
- После формулировки гипотез — перед планированием работ и приоритизацией.

Что уточнить у пользователя
- Ограничения по ресурсам/срокам, формат выдачи (epic/story/task), критерии готовности.

Выходной формат
```
Задача: <краткое действие>
Связь: гипотеза=<id|name>, субъект=<id|name|null>, цель=<id|name>
Тип: discovery|delivery|infra
Приоритет (предвар.): high|mid|low
```

JSON
```
{
  "tasks": [
    {"title": "...", "linked_hypothesis": "...", "goal_ref": "...", "subject_ref": "...", "type": "discovery|delivery|infra", "priority": "high|mid|low"}
  ]
}
```

Правила
- «Нет привязки — нет задачи»: всё должно быть присоединено к ветке карты.
- Набор задач отражает логику проверки гипотез; избегать «чёрных ящиков».

Подсказки
- Разбивка по направлениям (streams) и портфель/доска задач — как производные представления.
- Предлагай критерии готовности/результаты измерения (связь обратно с метриками цели).
