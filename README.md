# Набор скиллов для «Карты гипотез» (OpenCode‑ready)

Этот репозиторий содержит модульные скиллы, которые помогают быстро собирать и сопровождать стратегию в нотации «Карты гипотез»: цель, метрики, субъект и мотивация, гипотезы, задачи, приоритизация, экспорт. Скиллы готовы к автоподхвату в OpenCode и к дистрибуции как .skill‑пакеты.

## Зачем
- Унифицировать стратегические обсуждения: единая нотация и проверочные чеклисты.
- Ускорить фасилитацию: шаблоны вопросов и авто‑подсказки против типовых ошибок.
- Связать стратегию с планом работ: гипотезы → задачи → приоритетный маршрут.
- Делать «Стратегию как код»: структурированный экспорт и хранение в VCS.

## Кому полезно
- Продуктовым/проектным командам, PMO, коучам/фасилитаторам.
- Руководителям направлений, кто хочет упорядочить портфель инициатив.

## Состав модулей
- hm-hypothesis-builder — оформление и проверка гипотез (ядро)
- hm-goal-metrics — цели, метрики (текущие/целевые) и балансирующие метрики
- hm-subject-motivation — субъект, боли/желания, негативный субъект
- hm-task-linker — привязка задач к веткам, устранение «висящих» задач
- hm-review-troubleshooter — экспресс‑аудит по чеклистам и план фикса
- hm-topology-advisor — выбор топологий представления (доп.)
- hm-prioritization — приоритизация и приоритетный маршрут (доп.)
- hm-json-export — экспорт в JSON/YAML (доп.)

## Где лежат скиллы
- Для распространения/упаковки: `skills/<name>/SKILL.md` (+ `skills/shared/references/*`)
- Для автоподхвата в OpenCode: `.opencode/skills/<name>/SKILL.md`
- Разрешения: `opencode.json` (по умолчанию `allow` для всех скиллов)

## Быстрый старт в OpenCode
1. Откройте репозиторий в OpenCode. Автопоиск подхватит `.opencode/skills/*/SKILL.md`.
2. Проверьте доступные навыки: инструмент `skill` должен показывать наши `hm-*`.
3. Используйте естественные запросы (цели/субъекты/«если–то–потому что–тогда») — агент сам подгрузит нужный модуль.
4. При необходимости — явная загрузка: `use_skill("hm-hypothesis-builder")`.

## Упаковка и дистрибуция
- Готовые артефакты: `dist/*.skill` — zip‑упаковки каталогов скиллов.
- Собрать вручную (пример для одного модуля):
  ```bash
  zip -r dist/hm-hypothesis-builder.skill skills/hm-hypothesis-builder
  ```
- Публикуйте .skill как релизы или подключайте каталоги `skills/*` напрямую.

## Evals (черновые)
- `evals/evals.json` — 5 реалистичных кейсов для проверки построения гипотез и целей/метрик. Подходит для циклов тестирования/улучшения по методике skill‑creator.

## Структура репозитория (основное)
- `skills/` — исходники скиллов для дистрибуции
- `.opencode/skills/` — зеркала скиллов для автоподхвата OpenCode
- `dist/` — собранные `.skill`‑архивы
- `evals/` — примеры сценариев для проверки
- `opencode.json` — политика доступа к скиллам

## Синхронизация копий SKILL.md
Файлы дублируются намеренно (для упаковки и для автопоиска). Чтобы избежать рассинхронизации, можно добавить простой sync‑скрипт/CI, который копирует изменения из `skills/` в `.opencode/skills/`.

## Лицензирование
Методика «Карта гипотез» принадлежит автору. Для коммерческого использования методики уточняйте условия у правообладателя.
